# Contributing to documentation (feedback only)

Thank you for helping improve the PagoPA Developer Portal documentation (https://developer.pagopa.it).

This repository is a documentation source repository.
External contributions are collected through structured Issues and Discussions to ensure consistency, quality, and proper review.

## What you can do here
✅ Report documentation issues (typos, incorrect steps, outdated examples)  
✅ Suggest improvements (clarifications, additional examples, reorganization)  
✅ Report broken links or missing assets  
✅ Ask questions and request clarifications (via Discussions)

## What this repository does not support
❌ Pull Requests from external contributors  
❌ Technical support requests for products or environments

Documentation changes are reviewed and applied by the responsible editorial and product teams, in coordination with internal and external stakeholders.

## How to open an Issue
Go to Issues → New issue and select the appropriate template.

Required information typically includes:
- Product
- Page URL (Developer Portal or GitBook)
- Description of the issue or proposal
- Impact / priority

Screenshots and links are welcome.

## Issue lifecycle
Issues are managed using clear labels:

Status
- stato: nuovo
- stato: in revisione
- stato: in lavorazione
- stato: in attesa di chiarimenti
- stato: risolto
- stato: chiuso - non applicabile

Type
- tipo: errore
- tipo: miglioramento
- tipo: link rotto
- tipo: domanda

Priority
- priorità: alta
- priorità: media
- priorità: bassa

Product
- prodotto: io
- prodotto: send
- prodotto: firma-io
- prodotto: pdnd
- ...

## Discussions
For broader questions or topics not related to a specific page, please use Discussions.

## Privacy
Do not include personal data or sensitive information in Issues or Discussions.
