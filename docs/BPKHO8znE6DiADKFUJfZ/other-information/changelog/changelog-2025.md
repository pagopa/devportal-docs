# Changelog 2025

#### 1.0.134 (January 22)

* Added the possibility to modify the name of an e-service after its publication

#### 1.0.137 (March 3)

* Added further courtesy notifications

#### 1.0.138 (March 18)

* Added to the e-service card: e-service ID, e-service version ID, version publication date, interface file checksum, eventual suspension and deprecation date of the e-service version
* Added to the entity card IPA code / VAT number

#### 1.0.148 (June 3)

* Access to Consolidated Economic Account Companies (SCEC)

#### 2.0.0 (June 30)

* Delegations
* E-service Template
* Administrator required for Interop API client in case of write operations
* First API v. 2 release: endpoints already present in API v. 1 released
* DPoP, Distributed Proof of Possession

#### 2.1.0 (July 1)

* Introduced four new custom claims in the voucher issued by PDND Interoperability
* Introduced stringent constraints on claims for voucher issuance

#### 2.2.0 (August 4)

* [Second API v. 2 release](https://www.interop.pagopa.it/news/api-v2-secondo-rilascio): endpoints concerning usage requests, delegations, clients, entities, purposes, attribute creation and reading released

#### 2.5.0 (September 8)

* [Third API v. 2 release](https://www.interop.pagopa.it/news/api-v2-terzo-rilascio): endpoints related to e-services and dispenser keychains released

#### 2.6.0 (October 1)

* Management of Consolidated Economic Account Companies (SCEC) and Public Service Managers (GSP) attributes based on information in the IPA catalogue

#### 2.7.0 (October 6)

* [Fourth API v. 2 release](https://www.interop.pagopa.it/news/api-v2-quarto-rilascio): endpoints related to e-service templates released
* New sidenav: the side navigation menu in the front office is reorganized, with the aim of making the entries clearer and preparing the menu for the planned release of in-app notifications

#### 2.8.0 (November 18)

* [Indication on the processing of personal data for e-services](https://www.interop.pagopa.it/news/adeguamento-trattamento-dati-eservice)
* [Fifth API v. 2 release](https://www.interop.pagopa.it/news/api-v2-quinto-rilascio): management of e-service and e-service template attributes

#### 2.9.0 (November 27)

* [Facilitated purpose template](https://www.interop.pagopa.it/news/template-finalita): allows adherents to create pre-configured risk analyses for recurring use cases

#### 2.10.0 (December 3)

* [Sixth API v. 2 release](https://www.interop.pagopa.it/news/api-v2-sesto-rilascio): staying updated on state changes that occur on your objects on the platform through a polling mechanism (event channel — machine-to-machine events)
