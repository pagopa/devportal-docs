# Strumenti e riferimenti

## Guida alla nomenclatura degli e-service

La guida presenta **criteri di denominazione** e **buone pratiche** per descrizione, parole chiave e coerenza lessicale, così da rendere i servizi **ricercabili** e **comprensibili** nel catalogo, coerentemente con quanto previsto nell'allegato 7 delle nuove Linee Guida AgID.

🔗 [https://italia.github.io/pdnd-guida-nomenclatura-eservice/](https://italia.github.io/pdnd-guida-nomenclatura-eservice/)

## Validatore delle API

Strumento per **verificare** che i file **OpenAPI** siano **conformi** ai requisiti previsti (campi obbligatori, formati, coerenza). Si consiglia l’uso **prima** della pubblicazione di ogni versione.

🔗 [https://italia.github.io/api-oas-checker/](https://italia.github.io/api-oas-checker/)\
🔗 [https://github.com/italia/api-oas-checker-rules](https://github.com/italia/api-oas-checker-rules)

## Validatore semantico

Verifica la **correttezza dei metadati semantici** (vocabolari, riferimenti, coerenza) utilizzati nei file di interfaccia, a supporto dell’interoperabilità semantica.

🔗 [https://schema.gov.it/validatore\_metadati](https://schema.gov.it/validatore_metadati)

## Sviluppo di API conformi al ModI

Raccolta di **regole tecniche** e **pattern** per progettare API aderenti al **Modello di interoperabilità**: sicurezza, versionamento, gestione degli schemi, cicli di rilascio.

🔗 [https://developers.italia.it/it/news/interoperabilita.html](https://developers.italia.it/it/news/interoperabilita.html)\
🔗 [https://www.agid.gov.it/it/infrastrutture/sistema-pubblico-connettivita/il-nuovo-modello-interoperabilita](https://www.agid.gov.it/it/infrastrutture/sistema-pubblico-connettivita/il-nuovo-modello-interoperabilita)\
🔗 [https://www.agid.gov.it/it/argomenti/interoperabilita](https://www.agid.gov.it/it/argomenti/interoperabilita)\
