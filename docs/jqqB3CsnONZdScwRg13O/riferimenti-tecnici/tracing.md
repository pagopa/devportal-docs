# Tracing

### Cos'è?

Un servizio al quale inviare un report giornaliero del numero di chiamate API effettuate verso gli erogatori degli e-service dei quali il tuo ente è fruitore e ricevute dai fruitori per gli e-service che il tuo ente eroga.

Maggiori informazioni nel [webinar dedicato](https://developer.pagopa.it/webinars/DevTalks-pdnd-tracing).

### Obiettivi

Il servizio supporta gli aderenti nel dimensionamento dell’infrastruttura e consente a PDND Interoperabilità di monitorare l’andamento reale del traffico tra erogatori e fruitori, valutando l’efficacia complessiva della piattaforma.

### Come funziona?

Gli aderenti inviano a PDND Interoperabilità un tracciato giornaliero relativo alle chiamate API effettuate o ricevute. Il tracciato contiene esclusivamente:

* il numero di chiamate effettuate o ricevute;
* lo status code associato a ciascuna chiamata.

Maggiori dettagli operativi nella [guida dedicata](https://developer.pagopa.it/pdnd-interoperabilita/guides/manuale-operativo-tracing).

### È obbligatorio?

L'utilizzo del servizio è previsto nel ModI ed è poi ribadito nelle Linee Guida. Maggiori dettagli nella [sezione dedicata](../riferimenti-normativi/normativa-e-approfondimenti.md).
