# 📖 Introduzione al manuale

Questo manuale contiene una serie di informazioni create per i Service Provider (SP) come banche e istituti di pagamento, che desiderano implementare il servizio di Request to Pay (RTP) nei propri canali digitali.

***

### A chi si rivolge

Le linee guida sono destinate a tutte le figure coinvolte nella costruzione del servizio:

* **Designer e progettisti** che devono progettare l'esperienza di pagamento degli avvisi pagoPA e delle richieste di pagamento all'interno dei canali del Service Provider (es.: app di pagamento).
* **Project manager e figure strategiche** che devono individuare le opportunità che il servizio può portare sui propri canali.

### Perché seguire le regole

Queste regole sono la chiave per incoraggiare un maggiore utilizzo del servizio da parte degli utenti, e aumentare le probabilità che il pagamento avvenga sullo stesso canale in cui viene consegnata la richiesta di pagamento.

{% hint style="success" %}
**Queste linee guida nascono da una ricerca qualitativa approfondita.**

PagoPA ha condotto questo studio per capire le aspettative e le esigenze degli utenti sui pagamenti verso la Pubblica Amministrazione, specialmente per quanto riguarda la possibilità di ricevere le richieste di pagamento direttamente sui propri canali bancari.
{% endhint %}
