# Servizio @e.bollo 2.0

Le interlocuzioni tra Agenzia delle Entrate e PagoPA hanno evidenziato l’esigenza di studiare un framework evoluto per la gestione della vendita ed utilizzo delle Marche da Bollo.

Il servizio in essere attualmente presenta i seguenti limiti:&#x20;

* l’adozione della piattaforma pagoPA non è sufficiente all’integrazione con il servizio @e.bollo
* il sistema è complesso e poco utilizzato
* è disponibile unicamente il taglio da 16 euro
* sono solo 2 i PSP aderenti al servizio

L'evoluzione del servizio @e.bollo prevede l'implementazione dei seguenti use cases:

1. Pagamento spontaneo
2. Pagamento dovuto
3. Pagamento posticipato

Al momento è stato implementato soltanto lo use case numero 2 _**Pagamento dovuto**_.
