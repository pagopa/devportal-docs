# Servizio @e.bollo 2.0

Le interlocuzioni tra Agenzia delle Entrate ADE e PagoPA hanno evidenziato l’esigenza di studiare un framework evoluto per la gestione della vendita ed utilizzo delle Marche da Bollo.

{% hint style="info" %}
Tutti i dettagli relativi agli scenari previsti da ADE a cui si fa riferimento nel presente documento sono descritti all’interno della documentazione ufficiale scaricabile [qui](https://www.agenziaentrate.gov.it/portale/documents/20143/249463/Linee+guida+servizio+ebollo+per+scheda+-+citt_Ae.bollo_LineeGuida_release2.0_dic.18.pdf/57b3b3d5-2b7e-21cf-f77d-48498a344885?t=1551273230817).
{% endhint %}

L'evoluzione del servizio @e.bollo prevede l'implementazione del seguente scenario:

* **Scenario A - Pagamento dovuto** - Pagamento del bollo nell’ambito di servizi on line delle amministrazioni che generano “Documenti Informatici”.

{% hint style="info" %}
Il servizio **@e.bollo 2.0** non va in sostituzione del processo esistente [Servizio @e.bollo](../servizio-e.bollo.md), lo scenario attualmente implementato è da considerarsi come una semplificazione che può essere implementata discrezionalmente dall'EC.
{% endhint %}
