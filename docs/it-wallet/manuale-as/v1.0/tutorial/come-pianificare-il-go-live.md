# Come pianificare il go-live

Il go-live non si esaurisce in un'attivazione tecnica, ma richiede il coordinamento tra più soggetti. La definizione concordata della data con Issuer e Wallet Provider previene disallineamenti (ad esempio la disponibilità dell'attestato prima che il wallet sia in grado di mostrarlo); una comunicazione adeguata favorisce la comprensione, da parte degli utenti, delle funzionalità del nuovo EAA.

**Prerequisiti.** Esito positivo dei test in collaudo e in produzione.

## Passaggi

{% stepper %}
{% step %}
### **Concordare la data di rilascio**

La data di rilascio deeve essere concordata con l'Issuer e il Wallet Provider, affinché tutti gli attori risultino pronti nel medesimo momento
{% endstep %}

{% step %}
### <mark style="color:yellow;">**Effettuare la registrazione amministrativa**</mark>

<mark style="color:yellow;">Non appena sarà disponibile, dovrà essere effettuata la registrazione amministrativa, secondo quanto definito dal Regolamento IT Wallet. Allo stato attuale non costituisce un prerequisito: è un adempimento futuro che si perfeziona in prossimità del go-live</mark>
{% endstep %}

{% step %}
### Valutare le attività di comunicazione

Valutare l'avvio di un programma di comunicazione in coerenza con la **Brand Identity** del Sistema IT Wallet
{% endstep %}
{% endstepper %}

Successivamente al rilascio, l'attività prosegue con la gestione dell'EAA nel tempo, [descritta nel tutorial conclusivo](come-gestire-esercizio-manutenzione-e-assistenza.md).
