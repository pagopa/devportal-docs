# Riferimenti e strumenti a supporto

## Guida alla nomenclatura degli e-service

Il Dipartimento per la trasformazione digitale ha creato una guida ad hoc per la nomenclatura degli e-service, che consiste in una serie di buone pratiche per la loro nomina e descrizione su PDND Interoperabilità.

L'obiettivo è di rendere i servizi più facilmente ricercabili e identificabili.

Si richiede agli aderenti di attenersi alle indicazioni offerte nella guida, che è disponibile al seguente link: [https://italia.github.io/pdnd-guida-nomenclatura-eservice/](https://italia.github.io/pdnd-guida-nomenclatura-eservice/).

## Validatore delle API

Il Dipartimento per la trasformazione digitale ha pubblicato uno strumento che consente di verificare la compatibilità della propria interfaccia API REST rispetto a quanto richiesto nel ModI di AgID.

Si richiede agli aderenti di verificare che il proprio file di interfaccia passi il controllo del validatore prima di pubblicare il proprio e-service.

Il validatore delle API è disponibile al seguente link: [https://italia.github.io/api-oas-checker/](https://italia.github.io/api-oas-checker/), ed è corredato da [Guide all'utilizzo](https://github.com/italia/api-oas-checker-rules).

## Validatore semantico

Il Dipartimento per la trasformazione digitale, assieme a Istat, ha pubblicato uno strumento che consente di verificare i metadati delle risorse semantiche presenti nei propri file di interfaccia.&#x20;

Il validatore è parte di un più ampio progetto:

> Schema è il catalogo in continua evoluzione, che armonizza e standardizza i modelli di dati condivisi e garantisce che formato e significato delle informazioni scambiate siano preservati e compresi durante gli scambi.
>
> — [schema.gov.it](https://schema.gov.it/)

Il validatore semantico è disponibile al seguente link: [https://schema.gov.it/validatore\_metadati](https://schema.gov.it/validatore_metadati).

## Sviluppo di API conformi al ModI

Per alcuni articoli che trattano di costruzione di API, si vedano le [news dedicate](https://developers.italia.it/it/news/interoperabilita.html) su Developers Italia. Inoltre, si vedano la [pagina dedicata](https://www.agid.gov.it/it/infrastrutture/sistema-pubblico-connettivita/il-nuovo-modello-interoperabilita) al ModI sul sito di AgID, ed al relativo elenco di [aggiornamenti](https://www.agid.gov.it/it/argomenti/interoperabilita).

Per maggiori informazioni sui pattern di sicurezza, rimandiamo alle [Linee Guida AgID dedicate](https://www.agid.gov.it/it/agenzia/stampa-e-comunicazione/notizie/2023/05/23/aggiornati-i-pattern-sicurezza-linee-guida-sullinteroperabilita-tecnica-pa).&#x20;
