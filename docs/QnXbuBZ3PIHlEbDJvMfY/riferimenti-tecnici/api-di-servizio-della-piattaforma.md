# API di Servizio della Piattaforma

Oltre a orchestrare lo scambio di dati tra Erogatori e Fruitori, PDND Interoperabilità espone un proprio set di **API di servizio**. Queste API sono state progettate per permettere agli enti di automatizzare tutte le operazioni che altrimenti andrebbero eseguite manualmente tramite il back-office.

Utilizzando queste API, è possibile integrare la gestione di Interoperabilità direttamente all'interno dei propri sistemi gestionali, dei propri processi di CI/CD o di altri strumenti di automazione. Questo è particolarmente utile per i partner tecnologici o per gli enti che gestiscono un volume elevato di e-service, finalità o richieste di fruizione.

### Specifiche OpenAPI v2

Tutta la documentazione tecnica che segue fa riferimento esclusivamente alle **API v2**, che rappresentano la versione stabile e supportata della piattaforma. Le API v1 sono in dismissione e non devono essere utilizzate per nuove integrazioni.

Le API v2 coprono l'intero spettro delle funzionalità di gestione, tra cui:

* La gestione programmatica delle **richieste di fruizione** (agreements).
* La creazione e gestione delle **finalità** (purposes).
* L'interrogazione del **catalogo** degli e-service.

### Consultazione e Sandbox di Test

Le specifiche complete delle API di servizio, inclusi tutti gli endpoint, i modelli di dato, i parametri, i codici di errore e gli esempi, sono documentate secondo lo standard OpenAPI e sono disponibili in un'interfaccia web interattiva.

Questa interfaccia, non solo permette di navigare e comprendere in dettaglio ogni aspetto dell'API, ma include anche una **sandbox** che consente di testare le chiamate direttamente dal browser, semplificando il processo di sviluppo e integrazione.
